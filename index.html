About This Project

This project is a high-performance, production-grade web architecture designed to demonstrate the power of Infrastructure as Code (IaC) and Automated Deployment. By leveraging the AWS Global Infrastructure, this site ensures near-zero latency and high availability for users worldwide.

The entire environment was provisioned using Terraform, ensuring that the infrastructure is version-controlled, repeatable, and scalable. This represents a modern approach to cloud management where manual configuration is replaced by precise, executable code.

Deployment Status: ● Live via S3 & CloudFront OAC

Technical Stack

AWS CloudFront (CDN) Amazon S3 (Hosting & Lifecycle) S3 Bucket Versioning Terraform (IaC) GitHub Actions (CI/CD) IAM & OAC Security Route 53 & SSL

Data Resilience & Disaster Recovery (S3 Versioning)

To ensure high availability and protect the production environment against accidental deletions or malicious overwrites, I enabled Amazon S3 Bucket Versioning via Terraform.

Every time a new artifact or deployment is pushed through the CI/CD pipeline, S3 automatically keeps track of older iterations. This guarantees a robust Rollback Strategy—in the event of a faulty deployment, the website can be restored to a previous stable state within seconds, minimizing downtime and maintaining business continuity.

Cost Optimization & Automated Governance (S3 Lifecycle)

While versioning is critical for data safety, keeping infinite historical versions can lead to unnecessary storage costs. To counter this, I implemented an automated S3 Lifecycle Policy configured entirely through code:

  • Noncurrent Version Expiration: Historical or older versions of the website's files are automatically and permanently purged after 30 days, ensuring we only pay for what we reasonably need.
  • Multipart Upload Cleanup: Any incomplete or interrupted file uploads are automatically aborted and cleaned up after 7 days, eliminating hidden storage overhead.

This automated tiering aligns perfectly with the AWS Well-Architected Framework under the Cost Optimization pillar.

Automated CI/CD Pipeline (The Power of DevOps)

This website is integrated with a sophisticated Continuous Integration and Continuous Deployment (CI/CD) pipeline. The main advantage of this setup is the elimination of manual uploads and human error.

With GitHub Actions, any change I make to the code is automatically tested and synced to the AWS Cloud. I can update my entire global web presence with just three simple commands:

  • 1. git add .
  • 2. git commit -m "Update website"
  • 3. git push origin main

Once pushed, the pipeline handles the rest—invalidating the CloudFront cache and serving the fresh content to users globally within seconds.

Key Architecture Benefits

  • Security: Origin Access Control (OAC) ensures that the S3 bucket is completely private, making CloudFront the only gateway.
  • Performance: Content is cached at AWS Edge locations worldwide, reducing load times significantly.
  • Cost-Efficiency & Sustainability: Serverless architecture combined with strict lifecycle governance ensures zero waste and optimal pricing.
